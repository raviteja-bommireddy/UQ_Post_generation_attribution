Research Framework · Draft · For Mentor Review

Uncertainty Quantification in
Post-Generation Attribution

A structured methodology for measuring how stable evidence attribution is for LLM-generated factual claims — across four qualitatively distinct stages of the attribution pipeline.

Dataset: ASQA · 17 questions LLM: llama-3.1-70b · OpenRouter Retrieval: BM25 / Dense / Hybrid NLI: cross-encoder/nli-deberta-v3-base Perturbations: 12 per claim
Section 01

Framework Overview

UQ in Post-Generation Attribution is a measurement framework. We identify specific stages of the attribution pipeline and probe each for uncertainty.

Attribution uncertainty does not mean the claim is wrong. It means: how stably and consistently is this claim backed by retrievable evidence?

We isolate four stages — Attribution, Retrieval, Hallucination (Content), and Claim Extraction — and design targeted signals for each.

We Are Measuring
  • Stability of evidence entailment across perturbations
  • Consistency of retrieved document sets
  • Whether added specifics are grounded in the corpus
  • Consistency of claim decomposition across runs
We Are NOT Measuring
  • Correctness or truthfulness of the claim
  • Factuality verified against ground-truth
  • Model-internal token probabilities
  • Corpus completeness or quality
Why this is currently feasible: All components — BM25 retrieval, cross-encoder NLI, RBO computation — operate in a fully black-box setting. No model internals are needed. The pipeline runs on open datasets (ASQA, ALCE Wikipedia) with publicly available models.
Section 02

Identified Problems & Four UQ Stages

Post-generation attribution has no single point of failure. Uncertainty can enter at any stage. We map each to a distinct signal.

UQ₁ Attribution
→
Problem: Retrieved docs do not consistently entail the claim across surface variants. Signal: NLI entailment variance.
UQ₂ Retrieval
→
Problem: Different queries retrieve different documents — the evidence anchor is phrase-dependent. Signal: Rank-Biased Overlap instability.
UQ₃ Hallucination (Content)
→
Problem: LLM generates specific details not grounded in any retrievable passage. Signal: Retrieval Anchoring Bias + Specification Entailment Gap.
UQ₄ Claim Extraction
→
Problem: Atomic decomposition of the answer is unstable — different runs produce different claim boundaries. Signal: Under investigation.

Section 04

Retrieval & NLI Scoring — Shared Input to All UQ Stages

All 12 perturbations, plus the base claim, pass through the same retrieval and NLI pipeline. This produces the raw scores consumed by UQ₁, UQ₂, and UQ₃.

Base Claim fi + 12 Perturbations {P1…E2} → 13 queries total per claim
↓
BM25 or Dense or Hybrid Retrieval → top-k docs per query (k = 5)
↓
NLI Cross-Encoder (DeBERTa-v3) → P(entailment | doc, claim) per (doc, query) pair
↓
Score(fi) = max entailment over top-k docs → one attribution score per query
↓  {Score_base, Score_P1, Score_P2, …, Score_E2} → feeds UQ₁, UQ₂, UQ₃

The retrieval step can use BM25 (lexical), dense (embedding-based), or hybrid retrieval — the downstream UQ metrics are retrieval-method agnostic. All doc IDs and scores are stored per query for use in RBO and Jaccard computation.

Section 05
UQ₁ Attribution Uncertainty — NLI Entailment Variance
Core question: Do retrieved documents consistently entail the claim across all surface variants — and correctly fail to entail its corruptions?

Attribution score for a claim variant = max P(entailment | doc, variant) over the top-5 retrieved docs. Collected for all 13 variants (base + 12 perturbations).

all_scores = [Score_base, Score_P1, Score_P2, ..., Score_E2] # 13 values # A well-attributed claim should show: # HIGH entailment for Preserve perturbations (P1, P2) # LOW entailment for Destroy perturbations (D1, D2, D3, D4) # VARIABLE for specificity / breadth / depth perturbs UQ1_raw = Var(all_scores) # high variance → unstable attribution UQ1_norm = UQ1_raw / 0.25 # normalize: max theoretical Var = 0.25

High variance across all 13 scores indicates the NLI model cannot consistently find support — the claim has no stable evidence anchor. Low variance with correctly high/low scores by category indicates stable attribution.

Why NLI and not cosine similarity? Cosine similarity between embeddings measures topic proximity. NLI entailment directly measures "does this document support this claim?" — which is the definition of attribution. Cross-encoders (DeBERTa-v3) are significantly more precise than bi-encoders for this task.
Section 06
UQ₂ Retrieval Uncertainty — Evidence Set Instability
Core question: Do perturbations retrieve the same documents as the original claim? A claim may always score high on NLI — but from completely different documents across queries. That is retrieval uncertainty.

For every perturbation query, we record the ordered list of retrieved document IDs. We compare each perturbation's doc list against the base doc list using two metrics.

Metric A — Jaccard Similarity

Set-level overlap. Does not consider ranking order. J(A,B) = |A∩B| / |A∪B|.

A = {doc1, doc2, doc3, doc4, doc5} # base retrieval B = {doc1, doc6, doc3, doc9, doc5} # perturbation retrieval Jaccard(A,B) = |{doc1,doc3,doc5}| / |{1,2,3,4,5,6,9}| = 3/7 = 0.43
△ Stored as diagnostic — not primary signal

Metric B — Rank-Biased Overlap (RBO, Webber et al., 2010) — Primary Signal

RBO assigns exponentially higher weight to top-ranked positions. Document #1 matters far more than document #5. The persistence parameter p = 0.9 gives ~86% weight to the top-10 positions. This reflects how attribution pipelines (and humans) actually use retrieved docs — top first.

# RBO(S, T, p) = (1-p) × Σ_{d=1}^{∞} p^{d-1} × |S[:d] ∩ T[:d]| / d S = [doc1, doc2, doc3, doc4, doc5] # base (ranked) T = [doc1, doc6, doc3, doc9, doc5] # perturbation (ranked) # Depth-by-depth agreement: d=1: {doc1}∩{doc1} = 1 → X_1 = 1.0 (same #1 doc — high weight!) d=2: {1,2}∩{1,6} = 1 → X_2 = 0.5 d=3: {1,2,3}∩{1,6,3} = 2 → X_3 = 0.67 # ...etc. RBO ≈ 0.88 # high agreement — same top doc, mostly same set Instability = 1 - 0.88 = 0.12 # low retrieval uncertainty

Why not Jaccard as the primary signal?

Property Jaccard RBO (p=0.9)
Rank-sensitive ✗ No — rank #1 and rank #5 treated equally ✓ Yes — top positions weighted exponentially more
Reflects real retrieval usage ✗ No — set only ✓ Yes — top docs matter most in attribution
Captures ranking shift (same docs, different order) ✗ Misses this ✓ Catches this — RBO drops when order changes
Interpretability ✓ Simpler Moderate
Conclusion: Jaccard is stored as a secondary diagnostic. RBO is the primary retrieval UQ signal. UQ₂ (Instability) = 1 − mean(RBO over all 12 perturbation queries).
Section 07
UQ₃ Hallucination / Content Uncertainty — Two-Signal Approach
Core question: Did the LLM generate specific details that are not grounded in any retrievable corpus passage? We use two complementary signals and average them.

Signal A — Retrieval Anchoring Bias (Attribution Drop)

Measures whether the original claim phrasing enjoys disproportionately high support compared to its variants. If the base claim scores much higher than all its perturbations, the phrasing is over-anchored to specific corpus phrases — a potential hallucination indicator.

Score_base = NLI score for original claim Score_perturbs = [NLI scores for all 12 perturbations] Drop_raw = Score_base - mean(Score_perturbs) # Normalize signed value to [0, 1]: Signal_A = (Drop_raw - (-1)) / (1 - (-1)) # High Signal_A → base claim has inflated, phrase-specific anchoring # This is simple to compute — direct byproduct of NLI scoring step
△ Indirect — measures retrieval bias, not direct hallucination

Signal B — Specification Entailment Gap (SEG)

Directly uses Categories 4 & 5 perturbations (breadth + depth). If the base claim is well-entailed by the corpus but its specification variants (E1, E2: added causal/consequence details) are not entailed — those specific details were likely hallucinated.

Score_base = NLI score for original claim Score_specs = mean(NLI score for [E1, E2]) # depth/specification perturbs SEG = Score_base - Score_specs # how much do specs lose support? Signal_B = normalize(SEG, min=-1, max=1) # SEG ≈ 0 → specifics still entailed → content is grounded # SEG large → specifics not entailed → added details likely hallucinated
✓ More targeted — directly probes whether specific details are corpus-grounded
UQ₃ = (Signal_A + Signal_B) / 2 — We average both signals. Signal A is simple and provides a broad baseline. Signal B is more precise. Together they provide a richer hallucination content UQ than either alone.
Open question for UQ₃: The current approach is corpus-dependent. If the corpus simply does not contain a true specific detail, SEG will be high even though the claim is correct. A better future solution may involve LLM self-consistency sampling (SelfCheckGPT, Manakul et al. 2023) — repeatedly regenerating the answer at T>0 and checking NLI consistency. This is noted as a stronger alternative requiring further exploration.
Section 08
UQ₄ Claim Extraction Uncertainty — Open Problem
Research (HalluMeasure, 2024; FactScore, 2023) explicitly acknowledges that "the quality and consistency of claim decomposition directly impacts probe performance."

The way an LLM answer is split into atomic claims can introduce compounding errors into every downstream stage. Key failure modes:

  • Over-splitting: One fact split into two weak claims, each harder to individually attribute.
  • Under-splitting: Two distinct facts fused — forces multi-hop attribution that retrieval may not support.
  • Coreference loss: "He won the prize" — when "he" = Einstein, the extracted claim loses its entity anchor.
  • Boundary ambiguity: The LLM decomposer draws claim boundaries differently on each run — introducing non-determinism at the very start of the pipeline.
? Open Question

This stage is a crucial source of uncertainty — yet a principled, computationally feasible measurement strategy is still being developed. One candidate: run claim extraction K=3 times at T>0 and measure NLI agreement across runs. However, the right formulation of this uncertainty score, and how to integrate it with UQ₁–UQ₃, remains an open problem in this work.

Why not include in the composite score? Because UQ₄ measures uncertainty in the measurement process itself — meta-uncertainty. Including it in U_attr would conflate two qualitatively different things: attribution uncertainty and pipeline reliability. It is better maintained as a separate diagnostic.

Section 09

Composite Uncertainty Score — U_attr

The composite score combines UQ₁, UQ₂, and UQ₃ with user-supplied weights. Weights are provided at runtime — they are not hardcoded — reflecting the importance of each stage in the specific attribution setup being evaluated.

# Composite Attribution Uncertainty Score # All three signals normalized to [0, 1] before weighting. U_attr = w₁ × UQ1_norm ← Attribution: NLI variance across 13 variants + w₂ × UQ2_norm ← Retrieval: 1 − mean(RBO over 12 perturb queries) + w₃ × UQ3_norm ← Hallucination: avg(Signal_A, Signal_B) # Constraint: w₁ + w₂ + w₃ = 1.0 # Weights are provided by the user at runtime. # Suggested starting point: w₁=0.40, w₂=0.35, w₃=0.25 # UQ₄ (Claim Extraction) is stored as a SEPARATE DIAGNOSTIC. # It does NOT enter U_attr. Report it alongside U_attr for pipeline transparency. # Current implementation: UQ₁, UQ₂, UQ₃ only. # UQ₄ scoring methodology is under development.
Threshold interpretation (suggested):
U_attr < 0.2 → STABLE attribution  |  0.2 ≤ U_attr < 0.5 → MODERATE uncertainty  |  U_attr ≥ 0.5 → HIGH uncertainty
Section 10

Coverage Check — Are We Quantifying All Parts of Attribution Uncertainty?

Stage Source of Uncertainty Technique Status
UQ₁ · Attribution Evidence does not consistently entail the claim across surface variants of the same fact. NLI entailment variance (cross-encoder DeBERTa-v3) over 13 variants. ✅ Implemented. Best black-box method available.
UQ₂ · Retrieval Retrieved document identity shifts across perturbations — no stable evidence anchor. RBO (primary) + Jaccard (diagnostic). Instability = 1 − mean(RBO). ✅ Implemented. RBO is theoretically sound (Webber, 2010).
UQ₃ · Hallucination (Content) LLM generates specific details not grounded in retrievable corpus evidence. Avg(Retrieval Anchoring Bias, Specification Entailment Gap). Two-signal approach. ⚠️ Implemented. Better solution still sought — see open question in §07.
UQ₄ · Claim Extraction Atomic decomposition is unstable — different runs produce different claim boundaries, propagating error downstream. Not yet implemented. Candidate: multi-run NLI agreement. ❓ Open problem. Critical stage — best kept as separate diagnostic, not in U_attr.
Corpus Quality Corpus may not contain evidence for a true claim — causing false high uncertainty scores. Not measurable within the current pipeline scope. ⚪ Acknowledged limitation. Corpus completeness is an external assumption — cannot be surfaced as a UQ signal without a reference corpus.
Overall assessment: The pipeline covers three of four core stages with principled, implementable techniques. UQ₄ (Claim Extraction) remains the most important open problem — its impact propagates through every downstream measurement. Corpus quality is an acknowledged external assumption. UQ₃ (Hallucination) works but relies on corpus coverage; a self-consistency-based approach would strengthen it.
